# ClientHub Design System

## Brand Direction

ClientHub should feel like a calm B2B workspace for agencies and service teams. It should be credible, organized, and more human than a generic admin dashboard.

## Visual Principles

- Make client decisions visible.
- Show progress, files, and messages in one workspace.
- Use muted operational colors with a warm accent.
- Keep portal surfaces dense enough for real work.
- Avoid decorative cards that do not carry workflow information.

## Colors

- Background: `#F6F5F0`
- Foreground: `#18212A`
- Primary: `#273C57`
- Accent: `#B95738`
- Secondary: `#DBE7D3`
- Muted: `#E8EBE4`

## Components

- Hero: client workspace mockup with project progress.
- Project card: client, next step, status, progress.
- Portal overview: metrics plus project timeline.
- Project detail: milestones, files, messages.
- Case study: recruiter proof of protected B2B workflows.

## Responsive Rules

- Mobile uses single-column project and portal layouts.
- Desktop uses wide portal grids with timeline and secondary summary panels.
- Progress bars remain readable at all breakpoints.

## Accessibility Rules

- Strong contrast on muted backgrounds.
- Real links for navigation.
- Clear headings and labels.
- Avoid hover-only information.
